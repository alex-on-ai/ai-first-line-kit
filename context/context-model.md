# Company context model

> The single document that makes AI specific instead of generic. Fill every
> section with real, verified facts. Mark anything unconfirmed `⚠ UNVERIFIED`.
> The qualifier and the proposal writer both read this. A wrong context model
> here produces wrong output everywhere, so get it right and read it back.
>
> Everything below is a **template**. Replace the bracketed guidance. The inline
> examples (in _italics_) show the shape a real answer takes; delete them.

## 1. Identity

- **Company name:** [Your company]
- **One sentence about you:** [What you do, for whom, and the wedge that makes
  you different. _e.g. "We build custom software and AI features for businesses
  whose workflows have outgrown off-the-shelf tools, especially where the work
  touches money, compliance, or patient data."_]
- **What you are against:** [The lazy default you beat. _e.g. "Demo-day AI
  theater and template dev shops: claiming instead of showing."_]

## 2. Offer

- **Top services (1-3):** [...]
- **Entry offer (the smallest way to start with you):** [The low-risk first
  step. _e.g. "A two-week fixed-scope prototype sprint: a working prototype, an
  integration map, a delivery plan, and a build estimate."_]
- **Engagement models:** [T&M, fixed price, dedicated team, etc.]
- **Pricing:** [Keep this internal. The AI must never quote a number. Note only
  that pricing is fixed after a call.]

## 3. Ideal client (ICP)

- **Roles:** [Who signs. _e.g. "founders, operators, clinician-founders"_]
- **Company size:** [_e.g. "11-50 employees, established SMB or funded startup"_]
- **Region:** [_e.g. "US and EU"_]
- **Industries:** [Ranked. _e.g. "healthcare first, then SaaS and expert-led
  services"_]
- **Pains that signal fit:** [Concrete, specific. _e.g. "critical work breaking
  in handoffs between systems; a manual money-touching process losing revenue; an
  AI feature they cannot staff."_ Not "they want to grow."]
- **Buying triggers:** [Named signals. _e.g. "arrived with a specific named
  problem; a compliance deadline; new ops lead."_]

## 4. Disqualifiers (who you say no to)

[Be explicit; this is what lets the AI decline with a reason. _e.g.:_]
- Needs already covered well by an off-the-shelf product
- Frozen-spec "just code this" requests that skip discovery; pure staffing
- Hobby / student / B2C toy-app requests with no business behind them
- Budgets far below any custom-software floor

## 5. Proof (VERIFIED ONLY)

> The AI may cite **only** what is listed here, and use **only** these numbers.
> If a case is not for public naming, say so; the AI will anonymize it.

- **[Case name / or "Anonymous <industry> case"]:** [What you built.]
  Metrics (only if real and defensible): [_e.g. "40 percent lower admin
  workload, 25 percent operating-cost savings, 90 percent fewer manual errors"_]
  Naming: [public | keep anonymous]
- **[Case 2] ...**
- **Credentials:** [_e.g. "Top Rated on Upwork, 5.0 on Clutch"_ — only if true.]

## 6. Voice

- **Tone:** [_e.g. "confident, dry, exact; a senior builder explaining plainly."_]
- **Writing patterns:** [_e.g. "a number beats an adjective; show, don't claim."_]
- **Sticker phrases (optional):** [1-3 lines you actually use.]
- **How you never sound / forbidden words:** see `voice-guard.md`.
